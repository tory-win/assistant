토리 에이전트 컨테이너는 `launchd-code-services_default` 네트워크에도 붙어 `http://local-cliproxyapi:8321`로 LLM 게이트를 직접 호출한다. `host.docker.internal:8321`은 Mac/Docker Desktop의 IPv6/host-gateway/포트 바인딩 상태에 따라 `Errno 101 Network is unreachable` 또는 connection refused가 날 수 있어 정본 라우팅으로 쓰지 않는다. 즉시 처리 모델은 `claude-opus-4-6`을 1차로 쓰고, 429/5xx/연결 실패/빈 응답 시 `gpt-5.4`의 `/v1/chat/completions` 경로로 fallback 한다. GPT가 빈 응답을 주면 같은 GPT에 1회 재질문하고, 그래도 비면 도구 없는 최종 답변 강제 호출로 한 번 더 살린다. GPT가 `to=slack_read ...` 같은 텍스트형 도구 호출을 출력하면 사용자에게 게시하지 않고 내부에서 파싱·실행 후 최종 답만 다시 요구하며, 게시 직전에도 같은 패턴을 차단한다.

퇴근 모드(2026-06-12): 사용자가 비서 채널에 "퇴근" 또는 "퇴근했다"라고 말하면 워처가 `state/work-hours.json`에 다음 08:00 KST까지 offwork 상태를 기록한다. 컨테이너 entrypoint 는 offwork 동안 fetch 는 계속 돌려 feed/attention 을 쌓고, 사용자에게 보이는 `slack_brief`·`research_replier`·outbox 발송만 멈춘다. 08:00 KST 이후 자동 재개한다. 중간에 바로 재개하려면 비서 채널에 "출근" 또는 "출근했다"라고 말한다.

## 에이전트 팀·가독성 (2026-06-15)

비서 처리를 **분류 → 조사 → 포맷** 3역할 파이프라인으로 강화. 전부 가산(additive)·폴백 안전 — 어느 단계가 실패해도 직전 동작으로 degrade 한다.

- **분류기(triage) = 우선 라우터** — `tory_agent_llm.triage()`: 도구 없는 값싼 단발 LLM 이 보스 메시지를 `noise/todo/employee/followup/research/simple`로 분류. **이게 라우팅의 정본** — onwork/offwork 다음에 가장 먼저 돈다. employee/todo 즉답은 분류기가 그 경로로 볼 때만 탄다. (회귀 수정: 느슨한 직원조회 정규식이 '성낙천이 준 회의록 참고해 승인안 작성' 같은 조사요청을 가로채 엉뚱한 직원 todo 를 뱉던 버그 — 정규식을 triage 뒤로 내림.) 노이즈는 조사 없이 NO_REPLY, 진짜 요청만 조사로(다듬은 의도 주입). 어떤 오류든 `route=research`(또는 명백한 todo)로 폴백. 1차 결정적 게이트 `tory_format.is_noise()`가 명백한 노이즈를 LLM 전에 거른다(`ㅋㅋㅋ`→'' 버그 수정).
- **조사(research)** — `run_claude`/`tory_agent_llm` 4소스 도구루프(Slack·Gmail·Notion·Drive + 회사 메모리). max_iters 20→24. **finalizer**: 도구 루프가 소진되면 `"(부분 결과만)"` 문자열을 게시하던 대신 도구를 끄고 수집분으로 최종 답변을 강제 합성(완성 초안 + 못 찾은 건 `(확인 필요)`). DM 읽기 가능 — slack_read 설명에 'X가 나에게 준' 류는 conversations.list(types=im)+conversations.history 로 DM 을 읽으라고 명시(에이전트가 'DM 못 읽음'이라 잘못 단정하던 것 교정).
- **포맷(format)** — 답변은 **Block Kit**(헤더/섹션/구분선/컨텍스트, `tory_blocks.to_blocks`)으로 게시해 모바일 가독성↑, 블록 규격 실패 시 text 폴백(알림·접근성 위해 항상 text 동봉). 브리핑 메시지도 Block Kit(제목 헤더+섹션+legend·보드 푸터) — **완료 리액션·번호 legend 는 그대로**(리액션은 블록 메시지에도 동일).

**스레드 후속 지시 수신(핵심 수정)** — 워처가 **브리핑 스레드도 활성 등록**(`seed_brief_threads`, `slack-brief-state.brief_msgs` 기준)한다. 기존엔 '토리가 답한 스레드'만 따라가 보스가 브리핑 스레드에 단 후속 지시('3번 더 자세히 봐줘')를 놓쳤다. 이제 그 스레드 보스 답글을 받아, 브리핑 원문(번호↔항목)을 맥락으로 풀고 필요 시 `conversations.replies`로 직접 읽어 답한다.

**데일리 보드(캔버스)** — `slack-config.json` `use_canvas:true`면 브리핑이 **독립 캔버스** 보드를 동반 갱신하고 링크를 메시지에 단다(`tory_canvas`). 채널 탭 캔버스는 호출마다 탭이 누적되고 삭제 후에도 탭이 남으며 제거 API 가 user 토큰 스코프 밖(검증)이라 안 쓴다 — 독립 캔버스는 탭을 안 만들고 통삭제가 깨끗해 leak 없음(이전 보드 stored-id 통삭제+새로 생성, `state/canvas-board.json`). canvases:write 는 **user 토큰** 전용(봇 토큰엔 없음). 트레이드오프: 매 갱신마다 직전 보드를 지워 과거 브리핑의 '보드 열기' 링크는 죽는다(보드는 '오늘' 스냅샷).

**회귀 수정** — `render_prompt()`가 PROMPT_TMPL 누락 필드(todo_context 등)를 기본값으로 채운다. 이전엔 `tory_research_replier`가 3키만 넘겨 승인 조사(FULFILL)가 매번 `KeyError`로 죽던 회귀(06-12 todo-snapshot 도입분)를 수정.

## 불변식 (절대 보존)

1. **대외비 경계** — 회사 Slack/Google 유래 기억은 전부 user `awm_confidential`. 개인 `tory`·1129 UI 에 새면 사고다. harvest 는 feed kind(slack/google)에 `scope=company` 를 강제한다.
2. **발송 게이트** — 타인에게 가는 메시지는 *자동* 전송 금지(이 불변식은 유지). 토리의 *자발적* 발신은 비서 채널·셀프 알림뿐. **승인 게이트 발송(2026-06-15)**: `tory_send_gate.py` 가 '📤 발송 승인 요청'을 비서 채널에 올리고 보스가 ✅ 한 *그 건만* user 토큰(보스 명의)으로 원문 채널/스레드에 보낸다 — 보스 승인 없이는 절대 안 나간다(자동발송 아님, 멱등, TTL 7일). 리액션 판독은 reactions:read 가 없어 `conversations.history` 의 reactions 필드로 한다(reactions.get 금지). 엔진 LIVE. 게이트는 슬랙 발송뿐 아니라 **실행 액션**도 처리(`propose_action`+`_execute_action`): Gmail 초안(`gmail_draft`)·캘린더 일정(`calendar_event`, `tory_google_write.py`)·Notion 페이지 생성/추가(`notion_page`/`notion_append`, `tory_notion_write.py`) — 모두 보스 ✅ 후에만. 에이전트 도구: `propose_send`·`propose_gmail`·`propose_calendar`·`propose_notion`. Google 토큰은 `calendar.events`+`gmail.compose` 재인증(readonly 보존 → 브리핑 페처 무중단), Gmail 은 *초안만*(타인 메일 자동발송 회피). Notion 은 integration 'tory-read' 에 공유된 페이지에만 쓴다.
3. **읽기전용 Google** — scope 는 gmail/calendar/drive `.readonly` 3개만.
4. **secret** — 토큰은 `~/.hermes/.env`·`~/.torymemory/*.json`(600)에만, feed/기억엔 redact 후 적재.
5. **킬스위치** — `assistant/bin/tory-killswitch.sh` 가 폴러·브리핑·큐레이터를 즉시 정지(토큰 폐기는 수동 안내).

- **자료 시점 규칙**: 문서·PPT는 공유일을 보고 같은 주제 여러 버전이면 최신만 근거로, 오래된 건 '(YYYY-MM 기준)'으로 표시하고 현재 사실로 단정하지 않는다(`slack_files`가 공유일 표기).

- **새 메시지로 발송하되 삭제하지 않는다**(2026-06-11 — 브리핑은 히스토리로 쌓임). 완료 리액션만 반영되는 사이클은 재발송 없이 상태만 흡수. 리액션(번호·✅)은 **최근 브리핑 3개** 어디에 달아도 그 메시지의 번호 매핑으로 인식(`brief_msgs`). 새 항목·새 활동·리마인더 때만 새 메시지.
- Gmail 노이즈: 페처가 RFC2047 헤더 디코드 후 `~/.torymemory/mail-filters.json`(block_senders/block_subjects, 토리가 사용자 지시로 확장)으로 차단(2026-06-11). 후보는 사람 발신 + 결재 승인요청만.
- 변화 있을 때만 발송: 새 항목·새 활동·해결 반영(15분 묶음)·⏳리마인더(4h 대기, 08–22 KST).
- **완료 표시 = 리액션**: 번호 이모지(1️⃣…)=그 항목, ✅=전부. 같은 상대의 새 메시지가 오면 자동 재등장.
- **조사 리마인더 닫기 = 이모지**: 원문에 답할 필요가 없는 건 조사 보고/리마인더 메시지에 ✅·❌ 등 완료 이모지를 달면 `답장 불필요`로 닫는다. 조사 제안 부모 메시지의 ✅는 조사 승인 의미라 닫기 신호로 보지 않는다.
- 모든 시각 **KST 고정**(코드에 Asia/Seoul 박음 — 머신 tz 무관).
- 이미 답한 항목은 자동 해결(마지막 발화자=사용자 / Gmail 마지막 발신=me).
